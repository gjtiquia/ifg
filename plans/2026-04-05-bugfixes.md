## Bug 1: Scroll Indicator Number Formatting

### Location
`internal/ui/tui.go:146-172`

### Problem
The scroll indicator displays entry counts incorrectly for entries >= 100.

Current implementation:
```go
if n < 10 {
    scrollText += string(rune('0' + n))
} else {
    scrollText += string(rune('0' + n/10))
    scrollText += string(rune('0' + n%10))
}
```

For n=123:
- `n/10 = 12` → `rune('0' + 12)` = `'F'` (character code 70)
- `n%10 = 3` → `rune('0' + 3)` = `'3'`
- Result: "F3" instead of "123"

### Solution

Use `strconv.Itoa()` for proper integer-to-string conversion:

```go
import "strconv"

// Replace the number formatting block with:
scrollText := "["
scrollText += strconv.Itoa(state.ScrollOffset + 1)
scrollText += "-"
scrollText += strconv.Itoa(lastVisibleIdx + 1)
scrollText += " of "
scrollText += strconv.Itoa(len(state.Filtered))
scrollText += "]"
```

Or use `fmt.Sprintf`:
```go
scrollText := fmt.Sprintf("[%d-%d of %d]",
    state.ScrollOffset+1,
    lastVisibleIdx+1,
    len(state.Filtered))
```

Add tests in `internal/ui/tui_test.go`:

   - Create state with 100+ entries
   - Verify scroll indicator shows correct numbers

2. **TestScrollIndicatorWith1000Entries**
   - Create state with 1000+ entries
   - Verify scroll indicator shows "100-200" format correctly

## Bug 2: Double Terminal Restore

### Location
`main.go:102` and `main.go:111`

### Problem
`term.Restore()` is called twice:
1. Via defer on line 102
2. Explicitly on line 111

```go
defer term.Restore()  // line 102
// ...
term.Restore()  // line 111
```

This calls `screen.Fini()` twice, which may cause undefined behavior.

### Solution

Remove the explicit call on line 111, keeping only the defer:

```go
func main() {
    // ... flag handling ...

    term, err := ui.SetupTerminal()
    if err != nil {
        fmt.Fprintf(os.Stderr, "Error setting up terminal: %v\n", err)
        os.Exit(2)
    }
    defer term.Restore()

    // ... rest of function ...

    selectedCommand := runInputLoop(state, term)

    if selectedCommand != "" {
        fmt.Println(selectedCommand)
        os.Exit(0)
    }

    os.Exit(1)
}
```

The defer ensures `Restore()` is called regardless of how the function exits.

### Test Cases

No new tests needed - this is a cleanup fix. Existing behavior is unchanged.

| `main.go` | Remove explicit `term.Restore()` call, keep only defer |
